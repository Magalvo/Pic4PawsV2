# Claude Agent Instructions for Pic4Paws V2

## Working Method

Follow Spec-Driven Development (SDD). All features start from a work item, work spec, and failing test.

## Reasoning for Complex Work-Items

When implementing composition-heavy features (combining multiple clients/flows):
1. Reason through how flows compose existing clients
2. Analyze failure phase transitions
3. Plan result sanitization across layers
4. Design test matrix for phase combinations

This applies to work-items like:
- `PET-DRAFT-SAVE-FLOW-CLIENT-001`
- `PET-PUBLISH-FLOW-*`
- Any work-item composing ≥2 existing clients
