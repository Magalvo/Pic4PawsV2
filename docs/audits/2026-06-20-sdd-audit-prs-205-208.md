## 3. Deviations and Scope Creep

**None** for PRs #206, #207, #208 — all are tightly scoped to their work items.

**PR #205** has one process deviation: the branch is named `agent/MOBILE-ABRIGOS-PUBLIC-001`
and the commit message references the ID, but no `docs/work-items/MOBILE-ABRIGOS-PUBLIC-001-*.md`
document was ever created. The implementation (4 lines in `nav.ts` + 7 auth-guard tests)
correctly addresses the prior audit finding, but bypassed the SDD enriched-work-item requirement.

The `updating` variant in `WebPasswordConfirmViewModel` is listed in the work item spec
under "Confirm page states" (updating — password update in flight) but is never produced
by `createWebAuthUi`. The confirm page controls in-flight state via a separate `submitting`
boolean. The `updating` variant is therefore dead in both the boundary and the page.
This is a minor spec/implementation drift, not scope creep.
