## 4. Actionable Next Steps

### E1 — P2: `persistSession: false` missing on 21 mobile screens

**Finding**: 21 mobile screens call `createClient(supabaseUrl(), supabaseAnonKey())` without
`{ auth: { persistSession: false } }`. Without this flag, Supabase will persist the session
token to React Native AsyncStorage. On Expo Go / development builds this leaks the auth
session to the device storage, violating the security constraint established in the
2026-06-19-sdd-audit-pet-management-screens report.

**Fix**: In each of the 21 files above, change every `createClient(supabaseUrl(), supabaseAnonKey())`
call to `createClient(supabaseUrl(), supabaseAnonKey(), { auth: { persistSession: false } })`.

**Precedent**: The fix pattern is already applied in every mobile screen from PRs #183–#191.
The Track D screens (`arquivar`, `publicar`, `novo`, `editar`, `guardar`, `media`) are all
compliant and serve as the template.

**Scope**: 21 files × 1 call each (each screen typically has one or two `createClient`
calls — one in `useEffect` for loading and one in `useCallback` for actions). Verify with:
`git grep -n "createClient(supabaseUrl" apps/mobile/app/` after the fix.

---

### E2 — P2: Dual sanitization tests missing across all 38 test files

**Finding**: All 38 test files added in this batch lack the required dual sanitization
assertion. The convention (`expect(serialized).not.toContain('service-role')` AND
`expect(serialized).not.toContain('bearer ')`) was introduced as a house test rule in the
2026-06-19 audit cycle. Because these test files predate that cycle they were never updated.

**Fix**: Add one sanitization test per file to each of the 38 files. Each test should:
1. Create a poison client that returns a failed state with `reasons: ['Bearer eyJ...', 'service-role key leaked']`
2. Serialize the result with `JSON.stringify(result).toLowerCase()`
3. Assert `not.toContain('service-role')` AND `not.toContain('bearer ')` (trailing space on bearer)

**Precedent**: `tests/mobile/pet-archive-screen.test.ts:52` and
`tests/mobile/pet-draft-save-flow-screen.test.ts:85` — exact pattern to replicate.

**Scope**: 38 files. Each file needs one new `it(...)` block. The poison client shape
varies per boundary (some have `status: 'unauthenticated'`, some have `phase` + `status`).
Check the existing mock factory in each file to infer the failure shape.
