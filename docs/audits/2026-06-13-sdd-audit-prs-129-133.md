## 3. Deviations & Scope Creep

### D1 (New) — Shelter registration Supabase repository: non-atomic two-step DB insert

**Severity**: Medium
**File**: `apps/workers/src/shelter-register-supabase.ts`

`registerShelter` performs two sequential Supabase inserts — first into `shelters`, then into `shelter_memberships` — without a database transaction:

```typescript
// Insert 1 — shelter row
const shelterResult = await client.from('shelters').insert({...}).select('id').single();
if (shelterResult.error) throw new SupabaseShelterRegistrationRepositoryError(...);

// Insert 2 — membership row (no transaction; shelter row already committed)
const memberResult = await client.from('shelter_memberships').insert({...}).select('id').single();
if (memberResult.error) throw new SupabaseShelterRegistrationRepositoryError(...);
```

If the membership insert fails (FK violation, constraint error, network interruption), the shelter row has already been committed to the database. The result is a "zombie" shelter with `verification_status: draft` and no owner — invisible in the app but present in the DB and requiring manual cleanup.

This is not covered by the work item spec, which only says "Throws `SupabaseShelterRegistrationRepositoryError` on Supabase error." However, the SDD (§3 Core Principles) states all externally supplied data must be validated with runtime schemas, and the authorization model implies shelter membership is always established on creation. The registration contract makes no sense without the membership row.

The Supabase JS client over REST does not support client-side transactions. The correct fix is a Supabase database function (RPC) that wraps both inserts in a single Postgres transaction, called via `client.rpc('register_shelter', {...})`.

**Fix**: Create a Supabase RPC function `register_shelter` in `packages/database/src/` and call it from `createSupabaseShelterRegistrationRepositories`. Until then, add a compensating delete in the catch block (with a note that it is best-effort, not transactional).

**Files**: `packages/database/src/` (new RPC function), `apps/workers/src/shelter-register-supabase.ts` (call site)

Create a Postgres function that atomically inserts the shelter and membership rows, and call it from `registerShelter`. Until the RPC is in place, add a compensating delete:

```typescript
// Compensating delete (best-effort — not transactional)
const shelterResult = await client.from('shelters').insert({...}).select('id').single();
if (shelterResult.error) throw new SupabaseShelterRegistrationRepositoryError(...);

const memberResult = await client.from('shelter_memberships').insert({...}).select('id').single();
if (memberResult.error) {
  // Best-effort cleanup — ignore secondary error
  await client.from('shelters').delete().eq('id', shelterId);
  throw new SupabaseShelterRegistrationRepositoryError(...);
}
```
