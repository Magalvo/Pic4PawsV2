### P2 — Fix hardcoded `/media` path in client callsites

**File paths:**
- `apps/web/app/animais/page.tsx:25`
- `apps/web/app/animais/[petId]/page.tsx:46`
- `apps/mobile/app/(app)/(tabs)/animais/index.tsx:25`
- `apps/mobile/app/animais/[petId]/index.tsx:51`

**Problem:** All four callsites pass `mediaUrlPath: '/media'` as a literal string to
`createMediaUrlClient`. The MEDIA-URL-WORKER-001 work item introduced `WORKER_MEDIA_URL_PATH`
as an env-driven config key precisely so the path can be changed without touching source code.

**Fix:**

1. Add `NEXT_PUBLIC_WORKER_MEDIA_URL_PATH` to `apps/web/.env.example` (value: `/media`).
2. Export a `mediaUrlPath` helper from `apps/web/src/env.ts`:
   ```
   export const mediaUrlPath = (): string =>
     process.env.NEXT_PUBLIC_WORKER_MEDIA_URL_PATH ?? '/media';
   ```
3. Replace the four `'/media'` literals with `mediaUrlPath() as `/${string}`` in the
   respective page files.
4. For mobile, add `EXPO_PUBLIC_WORKER_MEDIA_URL_PATH` to `apps/mobile/.env.example` and
   a matching helper to `apps/mobile/src/env.ts`.

No new Worker route, client function, or migration needed. Low risk.

### P3 — Make `failDonation` failures observable

**Problem:** `apps/workers/src/donation.ts:371,378` call `failDonation().catch(() => {})`
which silently swallows any DB error from the cleanup step. If the donation row cannot be
marked as failed (e.g., transient DB error), the operation silently proceeds to return
502 without logging the secondary failure.

**Fix:** Replace `.catch(() => {})` with `.catch((err) => console.error('[donation] failDonation error:', err))`.
This preserves the existing behavior (the main 502 response is returned regardless) while
making the cleanup failure observable in Cloudflare Worker logs. No test changes needed.

No work item required — single-line change in `apps/workers/src/donation.ts`.
