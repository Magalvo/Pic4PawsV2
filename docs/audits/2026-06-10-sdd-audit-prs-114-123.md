## 3. Deviations & Scope Creep

**Scope creep (minor, beneficial):** Both `createWebPetStatusHistoryUi` and `createMobilePetStatusHistoryUi` expose a `getLoadingState()` factory method that the spec contracts do not mention. Consistent with good API design and the `loading` state is in the union; the financials boundaries add the `loading` union member without a factory. This asymmetry is harmless but worth noting as a cross-boundary inconsistency.

**Deviation 1 (test helper gap):** `tests/workers/pet-status-history.test.ts` `makeRepository` (lines 30тАУ36) is missing `getLifecycleEvents` from the base object literal, violating the `PetArchiveRepository` interface. The return type annotation `: PetArchiveRepository` should have been caught by TypeScript, but the `tests/` directory is excluded from the typecheck project references (see Deviation 2). No test fails today because `getLifecycleEvents` is never called in this file. However, any new test added here that uses `makeRepository()` without explicitly overriding `getLifecycleEvents` would hit a `TypeError: ... is not a function` at runtime instead of a clear assertion failure. The fix is one line: add `getLifecycleEvents: async () => [],` to the base object, matching the pattern in `tests/workers/pet-archive.test.ts:49`.

**Deviation 2 (typecheck blind spot):** The root `tsconfig.json` lists only `packages/` and `apps/` in its project references. Files under `tests/` are type-stripped by Vitest (esbuild) but never checked by `tsc`. Deviation 1 exists precisely because of this gap. This structural issue predates this audit scope but is newly observable because `PetArchiveRepository` grew its interface in PR #118 without the corresponding test helper update being caught.
